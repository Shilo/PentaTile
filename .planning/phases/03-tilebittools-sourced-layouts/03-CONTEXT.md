### Slot-table primary sources

- **D-74: `PentaTileLayoutBlob47Godot` slot table is sourced from BorisTheBrave's published 47-blob reference** at https://www.boristhebrave.com/permanent/24/06/cr31/stagecast/wang/blob.html. ### 8-bit Moore mask convention (NEW for PentaTile)

- **D-76: 8-bit Moore mask bit ordering: cardinal-anchored.** `N=1, E=2, S=4, W=8, NE=16, SE=32, SW=64, NW=128`. Edges in low nibble (matches existing `Wang2Edge` precedent N=1/E=2/S=4/W=8); corners in high nibble. Both Blob47Godot and TilesetterBlob47 use the SAME ordering — no per-layout bit-naming variants. Documented in each layout's class doc-comment.

- **D-77: `compute_mask` stays local per-layout.** No shared 8-bit Moore helper on `PentaTileLayout` base. Phase 2 precedent: every layout inlines its own compute_mask via `sample_fn.call(coord + offset)`. The ~10 lines of code per layout aren't worth expanding the base class API surface (which today is exactly the abstract virtuals + `_pack_alternative`).

- **D-78: 256→47 collapse via BorisTheBrave's algorithmic rule.** "A corner bit only matters if both adjacent edges are set." Implementation: `compute_mask` returns the raw 8-bit Moore mask; `mask_to_atlas` first applies the collapse rule (clear corner bits whose adjacent edges aren't both set), then dispatches via a 47-entry `const _MASK_TO_ATLAS` dict keyed on the COLLAPSED mask. The collapse function is total — every 8-bit input maps to exactly one of 47 valid configurations; no fallback slot needed. **Plan-phase MUST add a unit test** that enumerates all 256 masks, applies the collapse, and asserts every result hits a valid dict entry (catches any 47-entry dict transcription error).

### Mask 0 / out-of-table dispatch (Pitfall #9 instances)

- **D-79: `PentaTileLayoutTilesetterWang15` reserves a 16th 'stray fill' slot at a documented fixed coord.** Tilesetter's exported 5×3 atlas + the artist-authored stray fill produces effectively a 6×3 atlas. Plan-phase picks the canonical coord (suggest `Vector2i(5, 0)` — first cell past the 5×3 grid) and documents it in the layout's class doc-comment + bitmask PNG. `mask_to_atlas(0)` returns `Vector2i(5, 0)` (or whatever coord plan-phase locks). The bundled `penta_tile_layout_tilesetter_wang_15.png` includes the stray-fill slot pre-greyboxed.

- **D-80: 47-blob layouts (`Blob47Godot`, `TilesetterBlob47`) — `mask_to_atlas(0)` maps to the isolated-cell slot from the 47-blob configurations.** mask=0 (no neighbors) collapses naturally to the "lonely tile" configuration in BorisTheBrave's collapse table; that's a valid 47-blob slot. No reserved fallback needed; collapse rule covers all 256 inputs.

- **D-81: All Pitfall #9 dispatch lives in each layout's `mask_to_atlas`.** No layer-side null-handling is added. Phase 2's existing single-grid pipeline (commit `81813cd`'s mask=0 default-atlas dispatch) is preserved as the contract: every logic-painted single-grid cell receives a non-null atlas slot from its layout's `mask_to_atlas`.

### Single-grid vs dual-grid declaration

- **D-82: All 3 new layouts: `is_dual_grid()` returns `false` (single-grid).** Each layout dispatches at the logic cell with whole-cell silhouettes, matching the conventions' authoring models (BorisTheBrave's blob diagrams; Tilesetter's Set View). Pitfall #8 (logic-painted-only gate) and Pitfall #9 (mask=0 default dispatch) both apply per-layout.

- **D-83: Phase 2's single-grid pipeline reused as-is.** `PentaTileMapLayer._paint_via_layout`, `_compute_mask`, neighbor-affected re-render — all carry over unchanged. Plan-phase MUST audit the pipeline against 47-blob requirements specifically:
  1. **Neighbor-affected radius:** Phase 2's single-grid layouts sample ≤4 neighbors. 47-blob samples 8 (Moore). When a cell is painted/erased, the layer must re-render every cell whose mask depends on this one — for 8-Moore, that's the 8 neighbors of the changed cell. Verify the existing affected-cell loop covers diagonals; expand if not.
  2. **`mask=0` for an isolated 47-blob cell** is a valid configuration (lonely-tile slot). Verify Phase 2's mask=0 default-dispatch path doesn't short-circuit before the layer reaches a logic-painted 47-blob cell.
  3. **Erase semantics**: erasing a cell with 8 neighbors needs the layer to re-render up to 8 cells — match Phase 2's `_update_cells(coords, forced_cleanup=true)` invocation.

## Existing Code Insights

### Reusable Assets

- **`PentaTileLayout` base class** (`addons/penta_tile/layouts/penta_tile_layout.gd`) — Phase 3 layouts extend this directly. The base provides: `bitmask_template`, `description`, `_init` auto-load of `bitmask_template` from `_default_bitmask_template_path()`, `compute_mask`/`mask_to_atlas`/`is_dual_grid`/`resolve_display_strip`/`needs_synthesis` virtuals, `_pack_alternative` helper, `get_fallback_tile_set` default codegen from `bitmask_template` + `_fallback_atlas_grid_size()`. Phase 3 overrides exactly the same set Phase 2 layouts override. Adding a 47-blob layout is the SAME shape as Wang2Corner — just bigger atlas + 8-bit mask + 256→47 collapse.
- **`PentaTileAtlasSlot`** — return type for `mask_to_atlas`. Already supports `atlas_coords: Vector2i`, `transform_flags: int = 0`, `alternative_tile: int = 0`. Phase 3 layouts populate `atlas_coords` only (no rotation reuse — every blob/wang slot is hand-drawn).
- **`PentaTileMapLayer._paint_via_layout`** — single-grid logic-painted-only dispatch path (Pitfall #8 + #9 mitigations live here from Phase 2's commits 6553380..205fb67). Phase 3 reuses without modification, modulo D-87 audit.
- **`_generate_bitmasks.py`** — generator script. Helpers `draw_corner_mask`, `draw_edge_mask`, `draw_slot_outline` (no-op currently per Phase 2 fix), `new_atlas`, `gen_dual_grid_16`, `gen_wang_2_edge`, `gen_wang_2_corner`, `gen_minimal_3x3`. Phase 3 adds: a new 47-blob silhouette drawer; `gen_blob_47_godot`; `gen_tilesetter_wang_15`; `gen_tilesetter_blob_47`.

### Established Patterns

- **`@tool` + `class_name` + `extends PentaTileLayout`** at the top of every layout file.
- **`const _NEIGHBOR := Vector2i(...)` declarations** at file scope, named per the layout's bit convention (cf. Wang2Corner's `_NE/_SE/_SW/_NW`). Phase 3's 47-blob layouts add 8 such consts each (N/E/S/W/NE/SE/SW/NW).
- **`compute_mask` body**: `var mask := 0; if sample_fn.call(coord + _DIR): mask |= bit; ... return mask`. Phase 3's 47-blob compute_mask follows this with 8 lines.
- **`mask_to_atlas` body**: `if mask == 0: return <default-slot or null per layout>; var slot := PentaTileAtlasSlot.new(); slot.atlas_coords = ...; return slot`. Phase 3 layouts each branch on mask=0 first (D-79, D-80), then look up the collapsed mask in their `const _MASK_TO_ATLAS` dict.
- **`_default_bitmask_template_path()`** returns `"res://addons/penta_tile/layouts/penta_tile_layout_<slug>.png"`.
- **`_fallback_atlas_grid_size()`** returns the layout's atlas grid as `Vector2i(cols, rows)`. For Phase 3: `Vector2i(6, 3)` for TilesetterWang15 (5×3 + stray-fill column); `Vector2i(11, 5)` for TilesetterBlob47; `Vector2i(?, ?)` for Blob47Godot per BorisTheBrave's reference.
- **Class doc-comment (`##`)** summarizes the convention, mask bit numbering, dual-grid declaration, atlas layout, and source/reference. Phase 3 layouts include a "Slot table sourced from: [primary reference URL]" line.

### Integration Points

- **`PentaTileMapLayer.layout: PentaTileLayout`** @export property accepts any `PentaTileLayout` subclass — Phase 3 layouts plug in without changes to the layer.
- **`get_fallback_tile_set()`** flows through automatically once each layout overrides `_default_bitmask_template_path` + `_fallback_atlas_grid_size`. Phase 4 wires the consumer side; Phase 3 just exposes the shapes.
- **`addons/penta_tile/tests/comprehensive_bitmask_test.gd`** — Phase 3 layouts added to the layout list; pattern matrix covers them. New 47-blob-specific patterns added if existing 16 don't exercise 8-Moore mask edges (e.g. a 3×3 plus + diagonals to hit a corner-with-both-edges configuration).
- **No `.tres` files** ship for the new layouts in Phase 3 — fallback is codegen via `get_fallback_tile_set()` at runtime; demo-binding `.tres` files are Phase 5 territory if needed.

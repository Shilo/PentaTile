# Phase 8 Recommendation Disposition Matrix

Every major recommendation from the Phase 8 research set is dispositioned here. Dispositions use only: `accept`, `already covered`, `defer`, `reject`.

| Recommendation | Disposition | Requirement IDs / Existing Track | Evidence | Rationale |
| --- | --- | --- | --- | --- |
| Deterministic variation using stable per-cell hash and `TileData.probability` weights | accept | `VAR-01` | `.planning/REQUIREMENTS.md`; Godot `TileData` API docs; AGENTS.md variation determinism pitfall. | High user value and identity-compatible if implemented in `_update_cells()` without `randi()` or persistent caches. |
| PixelLab variation-bank pick instead of first-cell row-major pick | accept | `VAR-PIXEL-01` | Phase 3.5 local summaries; `README.md` PixelLab rows; PixelLab Create Tileset docs. | Completes the PixelLab value already shipped, but should share the same deterministic variation design as `VAR-01`. |
| Explicit top tiles for platformer caps | accept | `TOP-01` | `.planning/REQUIREMENTS.md`; PixelLab side-scroller primary docs mention top tile descriptions. | Valuable for side-scroller art. Must be explicit per mask/layout, never inferred from "tile below is filled". |
| Tilesetter Wang15 layout | defer | `TBT-01-DEFERRED`, `TEMPLATE-02-DEFERRED` | `.planning/REQUIREMENTS.md`; TileBitTools verifies Tilesetter template demand. | Keep backlog item, but primary slot-table source or user export sample is required before implementation. |
| Tilesetter Blob47 layout | defer | `TBT-02-DEFERRED`, `TEMPLATE-02-DEFERRED` | `.planning/REQUIREMENTS.md`; TileBitTools verifies Tilesetter template demand. | Same as Wang15: valuable ecosystem coverage, blocked on authoritative table/sample. |
| PentaBake fifth-archetype/converter tooling | defer | `TOOL-01` | `.planning/REQUIREMENTS.md` backlog. | Useful authoring tool, but not needed before visible runtime features; keep separate from core solver. |
| Wang/blob/single-tile to PentaTile converter | defer | `TOOL-02` | `.planning/REQUIREMENTS.md` backlog. | Useful for ecosystem import. Depends on clearer target package choice and should not become a compatibility shim. |
| Editor line/rect/bucket preview when `layout` is bound | defer | Phase 6 deferred editor-preview track | STATE pending todo; `.planning/research/editor-line-rect-preview.md`. | Real UX debt, but not part of Plan 08-01 and must stay scoped to preview, not inspector/editor-suite expansion. |
| Docs/distribution discoverability | already covered | `DOCS-06`, `DOCS-07`, `DOCS-08`, `DIST-01` | `07-SUMMARY.md`; README docs-site section; `DIST-01` backlog. | MkDocs/LLM docs are covered. Asset Library remains a distinct backlog item if adoption becomes the goal. |
| Benchmark-first performance work | accept | `PERF-02` | `.planning/REQUIREMENTS.md`; `05-LOC-AUDIT.md` hot-path audit. | Accept benchmarks and documented limits. Reject caches/shaders/worker managers until data shows need. |
| Multi-terrain metadata interop in one TileSet/public layer | accept | `MULTITERR-01..08`, `TERRAIN-01` later | `08-MULTI-TERRAIN-RESEARCH.md`; Godot `TileData` / `TileMapLayer` docs. | Accept Godot terrain metadata as authoring/indexing input while PentaTile keeps its deterministic solver. |
| Global solvers/backtracking | reject | Out of Scope / scope firewall | Better Terrain README; AGENTS.md identity guardrails. | This turns PentaTile into a general terrain solver and expands the hot path beyond current identity. |
| Godot terrain-solver delegation for generated visuals | reject | `MULTITERR-07` firewall | Godot `set_cells_terrain_connect()` docs; `08-MULTI-TERRAIN-RESEARCH.md`. | Built-in solver mutates cells/neighbors according to Godot's model, bypassing PentaTile layout dispatch. Metadata input remains accepted. |
| Terrains dock/editor wizard/bulk terrain-bit editor | reject | Out of Scope | Better Terrain and TileBitTools primary READMEs. | Belongs to editor tooling/terrain-framework scope. PentaTile uses typed exports and docs/templates instead. |
| Persistent coordinate caches | reject | Out of Scope / identity guardrail | `05-LOC-AUDIT.md`; AGENTS.md. | Directly conflicts with the "no persistent coordinate caches" identity constraint and is premature without `PERF-02`. |
| Parallel paint APIs beside `set_cell()` | reject | Out of Scope / identity guardrail | README current API; `05-LOC-AUDIT.md` TileMapDual comparison. | Defeats the native API win. Future scope must preserve normal `set_cell()` / `erase_cell()` painting. |
| Scriptable rule engines | reject | Out of Scope | Better Terrain comparison; AGENTS.md identity guardrails. | Too broad for PentaTile's layout-resource dispatch model; would create a framework, not a lightweight addon. |
| Metadata/entity spawning systems | reject | Out of Scope | Godot already has custom data and scene tiles; Phase 8 research triage. | Off-mission world-building/database scope. Not needed for autotile visuals. |
| Hex/isometric/grid-agnostic expansion | reject | Out of Scope | TileMapDual primary README; TileBitTools limitations. | Valid competitor differentiator, but PentaTile v0.3 should stay orthogonal and small. |
| GPU/procedural world generation | reject | Out of Scope; `PERF-02` prerequisite for any perf work | `.planning/REQUIREMENTS.md` performance notes. | Premature and outside demo-scale quality bar. Benchmark first. |
| Compatibility/migration tooling for v0.1/v0.2 breaks | reject | No-compat policy | AGENTS.md Breaking Changes Policy; README upgrading section. | Breaking changes are allowed; CHANGELOG/release notes are the only migration surface. No shims or schema/version markers. |
| Formal GUT suite as the next quality step | defer | `DIST-02` | `.planning/REQUIREMENTS.md`; current root `tests/` Godot-native harness. | Existing visual regression harness is the active bar. Formal suite can wait until distribution/adoption needs it. |
| RPG Maker A2/A4 subtile compositor | defer | `RPGM-01`, `RPGM-02`, `RPGM-03` | `.planning/REQUIREMENTS.md`; README external resources. | Real ecosystem value, but it is a separate quarter-tile compositor pipeline, not a Phase 8 scope decision. |
| External editor importers for Tiled/LDtk | defer | `IMPORT-01`, `IMPORT-02` | `.planning/REQUIREMENTS.md`. | Useful later. Do not promote until the v0.3 package chooses import/distribution as its focus. |
| Shader fallback for diagonal compositing | defer | `PERF-01` | `.planning/REQUIREMENTS.md`. | Potential optimization path only after `PERF-02`; do not introduce shader complexity speculatively. |

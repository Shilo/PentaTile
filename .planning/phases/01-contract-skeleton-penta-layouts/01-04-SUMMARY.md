## Verbatim Excerpts (acceptance proofs)

### atlas_contract setter (D-08 + PITFALLS §5)

```gdscript
@export var atlas_contract: TetraTileAtlasContract:
	set(value):
		if atlas_contract == value:
			return                                                                  # idempotence guard (D-08, PITFALLS §5)
		if atlas_contract != null and atlas_contract.changed.is_connected(_on_contract_changed):
			atlas_contract.changed.disconnect(_on_contract_changed)
		atlas_contract = value
		if atlas_contract != null:
			atlas_contract.changed.connect(_on_contract_changed)
		_queue_rebuild()
```

### _resolve_layout lazy singleton (D-07 / CONTRACT-04)

```gdscript
static var _DEFAULT_LAYOUT: TetraTileLayout = null

func _resolve_layout() -> TetraTileLayout:
	if atlas_contract != null and atlas_contract.layout != null:
		return atlas_contract.layout
	if _DEFAULT_LAYOUT == null:
		_DEFAULT_LAYOUT = TetraTileLayoutTetraHorizontal.new()
	return _DEFAULT_LAYOUT
```

### _paint_via_layout dispatcher (CONTRACT-03)

```gdscript
func _paint_via_layout(display_cell: Vector2i, layout: TetraTileLayout, source: int, sample_fn: Callable) -> void:
	_primary_layer.erase_cell(display_cell)
	_overlay_layer.erase_cell(display_cell)

	var mask := layout.compute_mask(display_cell, sample_fn)
	if mask == 0:
		return                                                                      # universal short-circuit (PITFALLS §4)

	var slot := layout.mask_to_atlas(mask)
	if slot == null:
		return
	_paint_with_slot(_primary_layer, slot, display_cell, source)
	_paint_overlay_for_slot(slot, display_cell, source)
```

## Decisions Made

- **Whole-file replacement instead of surgical edits.** The plan's `<action>` provided full file content. The v0.1 surface area being removed was scattered across the file (enum at line 6, export at 31-34, helper at 182-185, match at 108-152, constants at 11-24, etc.). A full overwrite is significantly more legible than a 30-edit surgical diff.
- **Refresh class_name registry mid-plan.** The first parse-check on the new file failed with `Identifier "TetraTileLayoutTetraHorizontal" not declared in the current scope` because `.godot/global_script_class_cache.cfg` hadn't been updated since the v0.1 baseline (it only knew about `TetraTileMapLayer`). Ran `godot --headless --import --path .` once to trigger a project scan that picked up all 5 new classes from Plans 02-03 + the 6th class from this plan. Subsequent parse-check exits 0.

## Deviations from Plan

### 1. Mid-plan godot --import to refresh the class_name registry

- **Found during:** Task 3.1 verification (parse-check failed)
- **Issue:** `--check-only` doesn't auto-rebuild the global class_name registry. Without it, the new file's references to `TetraTileLayoutTetraHorizontal` (Plan 03) and `TetraTileAtlasContract` (Plan 02) didn't resolve.
- **Fix:** Ran `godot --headless --import --path .` to trigger a project scan that registered all 5 new classes. The plan's parse-check command then succeeded.
- **Files affected:** `.godot/global_script_class_cache.cfg` (regenerated by the import; not committed — it's in .gitignore by Godot convention) + 5 `.uid` sidecar files (committed alongside the layer rewrite).
- **Acceptance impact:** None. The acceptance criterion "Godot --check-only exits 0" is met after the registry refresh.

**Total deviations:** 1 environmental fix-up. No functional deviation from the plan's verbatim file content.

## Issues Encountered

- **Class registry staleness.** Documented above as Deviation #1. Worth noting for future Godot-CLI workflows: Plans that add `class_name`-bearing files should explicitly trigger a project import before parse-checks that depend on cross-file class references.

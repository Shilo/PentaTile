## Pattern Assignments

### `addons/penta_tile/layouts/penta_tile_layout_pixel_lab_top_down.gd` (new layout subclass, single-grid + corner-mask + cached dispatch)

**Analog:** `addons/penta_tile/layouts/penta_tile_layout_blob_47_godot.gd` (single-grid + bundled PNG + const-dict dispatch + `_fallback_atlas_grid_size` override). Secondary analog for the corner-mask `compute_mask` body: `addons/penta_tile/layouts/penta_tile_layout_dual_grid_16.gd` (TL=1/TR=2/BL=4/BR=8 OR-fold — same convention PIXLAB uses per D-93).

**Why this analog:** Blob47Godot is the only existing single-grid layout with (a) a bundled greybox PNG, (b) an N×N atlas (N=7), (c) constant dispatch tables, (d) a `_fallback_atlas_grid_size` override returning a non-trivial Vector2i. PIXLAB swaps 7×7 → 8×8 and the const Dictionary → a runtime `Array[Vector2i]` cache built at `_init`. The `compute_mask` body is verbatim from DualGrid16 (NOT Wang2Corner — Wang2Corner samples diagonal cells with NE/SE/SW/NW = 1/2/4/8 bit semantics, which differs from PIXLAB's TL=1/TR=2/BL=4/BR=8 corner anchors).

**Header pattern** (lines 1–24 of analog Blob47Godot):
```gdscript
@tool
## Blob47Godot — 47-tile blob layout, 8-bit Moore mask, single-grid.
##
## Mask convention (D-76): N=1, E=2, S=4, W=8, NE=16, SE=32, SW=64, NW=128.
## NOT the canonical CR31 clockwise ordering — see _collapse_8bit_moore for
## the algorithm. The 256→47 collapse rule (D-78) per BorisTheBrave's reference
## ...
class_name PentaTileLayoutBlob47Godot
extends PentaTileLayout
```
**Change for PIXLAB:** keep `@tool` + multi-line `##` doc-comment style (anchors to D-94/D-89/D-92/D-104). Replace `class_name` with `PentaTileLayoutPixelLabTopDown`. Doc-comment must cite spike 003 + `tileset_transform.lua:17-26` for the cell-to-role table source, and explicitly call out D-89 ("first-cell row-major pick; bank pick deferred to v2 VAR-PIXEL-01"). Mention D-104 mask=0 → role 12 → cell (2, 2) for top-down.

**Direction-offset constants pattern** (analog Blob47Godot lines 26–33; DualGrid16 lines 20–23):
```gdscript
const _N := Vector2i(0, -1)
const _E := Vector2i(1, 0)
const _S := Vector2i(0, 1)
const _W := Vector2i(-1, 0)
const _NE := Vector2i(1, -1)
...
```
**Change for PIXLAB:** Use the 4-corner subset matching DualGrid16 (`_TL`, `_TR`, `_BL`, `_BR`) — drop `_N/_E/_S/_W` and the diagonals. Verbatim from `penta_tile_layout_dual_grid_16.gd:20-23`:
```gdscript
const _TL := Vector2i(-1, -1)
const _TR := Vector2i(0, -1)
const _BL := Vector2i(-1, 0)
const _BR := Vector2i(0, 0)
```

**Const dispatch table pattern** (Blob47Godot lines 35–57):
```gdscript
# 47 entries keyed on D-76-ordered, COLLAPSED masks → atlas (col, row) coords
# in the 7×7 canonical packing. ...

      0: Vector2i(0, 0),    1: Vector2i(1, 0),    2: Vector2i(2, 0),    3: Vector2i(3, 0),
      ...

```
**Change for PIXLAB:** Replace const Dictionary with two const Array[int] tables and one runtime `var Array[Vector2i]` cache:
```gdscript
# 64 ints, row-major (row * 8 + col). Verbatim from
# tileset_transform.lua:17-26 `tileset_output`. Each int is a role-ID 0..15.
const _CELL_TO_ROLE := [
    6, 6, 6, 6, 6, 6, 6, 6,
    6, 7, 9, 10, 7, 9, 10, 6,
    6, 11, 12, 8, 15, 12, 1, 6,

const _ROLE_TO_MASK := [4, 10, 13, 12, 9, 14, 15, 7, 2, 3, 11, 5, 0, 8, 6, 1]
```

**`_init` + cache-build pattern** (analog: base `penta_tile_layout.gd:35-43`; new pattern not present in Blob47Godot since blob uses parse-time const):
```gdscript
# Base class _init (lines 35-43) — auto-loads bitmask_template:

    if bitmask_template != null:
        return
    var path := _default_bitmask_template_path()
    if path.is_empty():
        return
    var tex := load(path) as Texture2D
    if tex != null:
        bitmask_template = tex
```
**PIXLAB pattern (new — derived from D-97 + Pattern 2 in RESEARCH.md):**
```gdscript
func _init() -> void:
    super()  # base class auto-fills bitmask_template if null
    _init_cache()


func _init_cache() -> void:
    _first_cell_by_mask = []
    _first_cell_by_mask.resize(16)
    for i in 16:
        _first_cell_by_mask[i] = Vector2i(-1, -1)
    for row in 8:
        for col in 8:
            var role: int = _CELL_TO_ROLE[row * 8 + col]
            var mask: int = _ROLE_TO_MASK[role]
            if _first_cell_by_mask[mask] == Vector2i(-1, -1):
                _first_cell_by_mask[mask] = Vector2i(col, row)
```
The "only set if unfilled (Vector2i(-1, -1))" guard is the row-major first-cell rule per D-89 — mirrors PixelLab's own `tileset_transform.lua:73` ("Only store first occurrence").

**`is_dual_grid` + `compute_mask` pattern** (verbatim from `penta_tile_layout_dual_grid_16.gd:26-36`):
```gdscript
func is_dual_grid() -> bool:
    return false  # PIXLAB: false; DualGrid16 returns true. Per D-96 + LAYOUT-09.


func compute_mask(coord: Vector2i, sample_fn: Callable) -> int:
    var mask := 0
    if sample_fn.call(coord + _TL): mask |= 1
    if sample_fn.call(coord + _TR): mask |= 2
    if sample_fn.call(coord + _BL): mask |= 4
    if sample_fn.call(coord + _BR): mask |= 8
    return mask
```

**`mask_to_atlas` pattern** (Blob47Godot lines 77–88):
```gdscript
func mask_to_atlas(mask: int, _strip_index: int = 0) -> PentaTileAtlasSlot:
    var collapsed := _collapse_8bit_moore(mask)
    var slot := PentaTileAtlasSlot.new()
    slot.atlas_coords = _MASK_TO_ATLAS.get(collapsed, Vector2i(0, 0))
    slot.transform_flags = 0
    slot.alternative_tile = 0
    return slot
```
**Change for PIXLAB:** Drop the `_collapse_8bit_moore` step (PIXLAB's mask is already 4-bit; no collapse). Replace dict `.get()` with cache lookup. **DO NOT add `if mask == 0: return null`** (Pitfall #9 / D-104 — the cached `_first_cell_by_mask[0]` returns role 12's first cell which is a valid render target).
```gdscript
func mask_to_atlas(mask: int, _strip_index: int = 0) -> PentaTileAtlasSlot:
    # D-104: mask=0 dispatches to role 12's first cell (top-down (2,2)).
    # Per Pitfall #9, single-grid mask=0 is NOT erase.
    var slot := PentaTileAtlasSlot.new()
    slot.atlas_coords = _first_cell_by_mask[mask]
    slot.transform_flags = 0
    slot.alternative_tile = 0
    return slot
```

```gdscript
func _default_bitmask_template_path() -> String:
```

**LOC budget:** ≤ 70 LOC per D-105 (Blob47Godot is 112 LOC including the collapse function — PIXLAB drops the collapse so projected LOC ~58–65, well under budget).

### `tests/pixellab_first_cell_test.gd` (new unit-style test, locks D-89 first-cell pick contract)

**Analog:** `tests/blob_47_collapse_test.gd` (single-script SceneTree harness, asserts per-mask dispatch contract, hand-derived expected table, mask=0 explicit assertion).

**Why this analog:** Both tests assert "for each mask in [0..N), the layout's `mask_to_atlas(mask).atlas_coords` returns the correct cell."         if not dict.has(collapsed):
```gdscript
            _failures.append("raw mask %d → collapsed %d not in _MASK_TO_ATLAS" % [raw, collapsed])

    elif slot0.atlas_coords != Vector2i(0, 0):
        _failures.append("mask_to_atlas(0).atlas_coords=%s; expected Vector2i(0, 0) per D-80 lonely-tile dispatch" % slot0.atlas_coords)
    elif slot0.transform_flags != 0:
        _failures.append("mask_to_atlas(0).transform_flags=%d; expected 0 — D-77 forbids rotation reuse for blob layouts" % slot0.transform_flags)
```

- Expected table is a `Dictionary` of `mask → Vector2i` for EACH subclass (16 entries each, hand-derived from the row-major scan of `_CELL_TO_ROLE`).
- Coverage loop is `for mask in range(16)` (not `while raw < 256` collapse coverage).
- mask=0 assertion: top-down expects `Vector2i(2, 2)`; side-scroller expects `Vector2i(0, 0)` (D-104).
- `transform_flags == 0` assertion: anchor to D-90 ("no rotation reuse for PIXLAB") instead of D-77.
- Per RESEARCH Example 2: the hand-derivation IS load-bearing — plan-phase derives full 16-entry expected table for both subclasses; executor types in. Verify-the-regression cycle (CLAUDE.md Test Methodology #5): stash the `_init_cache` body, rerun, confirm test fails.

**Per RESEARCH.md Pitfall 2:** MUST verify both subclasses independently — top-down and side-scroller share the bijection but have different `_CELL_TO_ROLE` tables, so first-cell coords differ for almost every mask. Test failure for one subclass but not the other = mistakenly cross-referenced cell-to-role.

```python
def draw_corner_mask(draw: ImageDraw.ImageDraw, col: int, row: int, mask: int) -> None:
    """Fill quadrants of slot (col, row) per a corner mask.

    Bits: TL=1, TR=2, BL=4, BR=8.
    """
    x0, y0 = col * TILE, row * TILE
    half = TILE // 2
    quads = [
        (1, x0, y0, x0 + half - 1, y0 + half - 1),               # TL
        (2, x0 + half, y0, x0 + TILE - 1, y0 + half - 1),         # TR
        (4, x0, y0 + half, x0 + TILE - 1, y0 + half - 1),         # BR (typo in source — but layout: bottom-right)
        (8, x0 + half, y0 + half, x0 + TILE - 1, y0 + TILE - 1),  # BR
    ]
    for bit, qx0, qy0, qx1, qy1 in quads:
        if mask & bit:
            draw.rectangle((qx0, qy0, qx1, qy1), fill=GREY)
```

### D-101 Silhouette Switch (Documented Observed Behavior)

Plan 04 closeout records that the bundled-PNG silhouette convention switched from D-101 option A (per-role corner-quadrant fill) to option B (solid 32×32) after `comprehensive_bitmask_test` exposed a single-grid solidity violation: option A's partial-quadrant silhouettes left isolated cells rendering at 25-75% coverage instead of 100%. Same convention as Wang2Corner + Blob47Godot — single-grid layouts encode mask via atlas POSITION (which cell first-cell-pick selected), not silhouette shape.

**Status:** Documented expected fallback path (PATTERNS.md anticipated it; D-101 names option A as default + option B as fallback). NOT a deviation — explicitly within the plan's allowed solution space.

**Evidence:**
- `bitmask_bounds_test.gd` lines 87-93 inline comment documents the switch with rationale.
- `_generate_bitmasks.py` lines 322-345 `draw_pixel_lab_cell` ships the option-B solid silhouette.
- `03.5-04-SUMMARY.md` line 21 records `affects: ... draw_pixel_lab_cell switched D-101 option A → option B`.
- Both bundled PNGs regenerated with the option-B convention; `bitmask_bounds_test` and `comprehensive_bitmask_test` both green.

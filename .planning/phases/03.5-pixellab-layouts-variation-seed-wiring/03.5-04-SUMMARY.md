## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] D-101 option A → option B fallback (planned alternate path)**

- **Found during:** Task 4.2 (running comprehensive_bitmask_test for the first time after adding PIXLAB layouts to the matrix list)
- **Issue:** Single-grid solidity assertion (Assertion 2 in comprehensive_bitmask_test) requires every painted cell to dispatch to a 100%-solid atlas tile. Option A (role-coded corner-mask silhouettes) produced 25%-75% pixel coverage for masks 0/1/2/4/8/etc., so 72 combos failed across both PIXLAB layouts × all 18 patterns. The role-coded silhouette is a cosmetically-nicer inspector preview but cannot survive single-grid rendering: an isolated painted cell renders one atlas tile alone, with no neighbor to compose with, so non-100% silhouettes leave visible gaps.
- **Fix:** Switched `_generate_bitmasks.py` `draw_pixel_lab_cell` to a 1-line solid-32×32 fill (`draw.rectangle(...)`); regenerated both PIXLAB PNGs; updated `bitmask_bounds_test`'s PIXLAB silhouette callers to reuse the existing `_solid_silhouette` helper instead of introducing PIXLAB-specific helpers. The pre-emptive PIXLAB-specific cell-to-role tables that the plan instructed me to add to `bitmask_bounds_test.gd` are no longer needed — removed. This matches the PATTERNS.md anticipated fallback path verbatim ("Per RESEARCH Pitfall 6: option A first try ... fall back to solid 32×32 ... only if `pixellab_visual_regression_test` reports gap regressions"). The pivot fired one plan early — the matrix test caught what the visual regression test would have caught later. Same UAT lesson Wang2Corner learned in Phase 2 (gen_wang_2_corner docstring lines 234-251): single-grid layouts encode mask via atlas POSITION, not silhouette shape.
- **Files modified:** `_generate_bitmasks.py` (draw_pixel_lab_cell body), 2 PIXLAB PNGs (regenerated), `bitmask_bounds_test.gd` (PIXLAB callers + helpers removed)
- **Commit:** cb740b9 (rolled into Task 4.2)

The fix is consistent with the Test Methodology #6 trace-the-pipeline discipline: I traced from the failure (25% solidity at atlas (4,3)) → through `mask_to_atlas` (cache returns (4,3)) → through `compute_mask` (`_BR=(0,0)` reads self) → through the silhouette generator (option A draws partial quadrants for non-15 masks) → arrived at the planned fallback before patching. The compute_mask `_BR=(0,0)` self-sample was NOT touched — that's a property of dual-grid corner sampling that PIXLAB inherits per spike 003's locked role-to-mask convention; the silhouette is the right abstraction layer to fix.

**CREATE `tests/terrain_autodetect_test.gd`** — validates auto-detection core per D-16 item 1 and RESEARCH.md §Validation Architecture:

Test must validate:
1. **No TileSet** — `_auto_detect_terrains()` on a fresh layer with null tile_set is a no-op (no crash, no terrains created).
2. **Single source, 3 rows** — Create a TileSet with one atlas source (32×32 tiles, 4 cols × 3 rows). Set `layer.tile_set = ts` → verify 3 terrains created in terrain_set 0 with names "Terrain 0", "Terrain 1", "Terrain 2".
3. **TileData.terrain assignment** — After auto-detection, read `tile_data.terrain` for tiles at (0,0), (0,1), (0,2), (3,0), (3,1), (3,2). All should equal their atlas_coords.y value (0, 1, 2 respectively).
4. **terrain_set mode** — Verify `ts.get_terrain_set_mode(0)` matches the layout's `terrain_mode()` return (MATCH_CORNERS for DualGrid16).
5. **Snapshot-before-clear** — Rename "Terrain 0" to "Floor" via `ts.set_terrain_name(0, 0, "Floor")`. Reassign the same TileSet (or trigger auto-detection again). Verify terrain 0's name is still "Floor" (not overwritten to "Terrain 0").
6. **Multi-source stacking (D-15)** — Add a second atlas source with 2 rows. Verify total terrains = 3 + 2 = 5. Verify terrain IDs 3-4 from second source.
7. **Empty source skipped** — Source with 0 rows or no tiles → not counted in terrain total.

func _build_atlas_tileset(rows: int, cols: int, tile_size: int = 32) -> TileSet:

		for col in range(cols):
			src.create_tile(Vector2i(col, row))
	return ts

## Decisions Made

- **Recursion guard placement:** Added `_performing_auto_detect` check at `_auto_detect_terrains()` entry plus reset at all return points, rather than relying solely on the disconnect-reconnect pattern in `_on_tile_set_changed` (which didn't cover the deferred call path from `_set`)
- **Variation test layout:** Used Wang2Edge (single-grid) for determinism variation test because dual-grid path in `_paint_via_layout` does not route through `_pick_variation_tile` — variation only applies to single-grid layouts
- **Type annotations:** Used explicit `var used: Array` instead of `:=` for `get_used_cells()` results — GDScript 2 cannot infer types from some TileMapLayer method returns in headless mode

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed `get_tree().process_frame` parse error in headless scripts**
- **Found during:** Task 1
- **Issue:** `await get_tree().process_frame` causes parse error "Function get_tree() not found in base self" in Godot 4.6.2 headless SceneTree scripts
- **Fix:** Replaced all `await get_tree().process_frame` with `await process_frame` (bare signal available on SceneTree)
- **Files modified:** tests/terrain_autodetect_test.gd
- **Verification:** Test compiles and runs — ALL PASS

**2. [Rule 1 - Bug] Fixed `get_resource_uid()` nonexistent on TileSetAtlasSource**
- **Found during:** Task 1
- **Issue:** `src.get_resource_uid()` doesn't exist on TileSetAtlasSource in Godot 4.6.2
- **Fix:** Used `ts.add_source(src)` without second argument (atlas_source_id_override defaults to -1)
- **Files modified:** tests/terrain_autodetect_test.gd
- **Verification:** ALL PASS with correct source IDs

**3. [Rule 1 - Bug] Fixed infinite recursion in `_auto_detect_terrains`**
- **Found during:** Task 1
- **Issue:** `_set` hook connects `tile_set.changed` signal THEN calls `_auto_detect_terrains.call_deferred()`. When deferred runs, it mutates TileSet → `tile_set.changed` fires → `_on_tile_set_changed` calls `_auto_detect_terrains()` again → infinite loop
- **Fix:** Added `if _performing_auto_detect: return` at `_auto_detect_terrains()` entry, set `_performing_auto_detect = true` at start, reset at all return points
- **Files modified:** addons/penta_tile/penta_tile_map_layer.gd
- **Verification:** All 7 sub-tests pass, no hang

**4. [Rule 1 - Bug] Fixed GDScript 2 type inference failures on `get_used_cells()`**
- **Found during:** Task 2
- **Issue:** `var used := visual.get_used_cells()` fails parse with "Cannot infer the type of 'used' variable" in headless mode
- **Fix:** Changed to explicit type annotations: `var used: Array = visual.get_used_cells()`, `var cell_count: int = ...size()`
- **Files modified:** tests/terrain_dispatch_test.gd, tests/terrain_determinism_test.gd, tests/terrain_sample_terrains_test.gd
- **Verification:** All 3 tests parse and run successfully

**5. [Rule 1 - Bug] Fixed variation mode not applied to DualGrid16 layout**
- **Found during:** Task 2
- **Issue:** `test_different_seed_different_output` returned 0 cell differences because DualGrid16 uses the dual-grid render path which doesn't call `_pick_variation_tile`
- **Fix:** Switched test to use Wang2Edge (single-grid) with `variation_mode = PROBABILITY`. Documented that variation only applies to single-grid layouts
- **Files modified:** tests/terrain_determinism_test.gd
- **Verification:** 6 cells differ between seeds 42 and 99

**6. [Rule 1 - Bug] Fixed sample_terrains.png loading — `load()` returns Texture2D not Image**
- **Found during:** Task 2
- **Issue:** `load("res://...") as Image` returns null for PNG files; Godot imports PNGs as CompressedTexture2D
- **Fix:** Used `load().get_image()` to extract Image from Texture2D
- **Files modified:** tests/terrain_sample_terrains_test.gd
- **Verification:** 3 terrains auto-detected from 160×96 image

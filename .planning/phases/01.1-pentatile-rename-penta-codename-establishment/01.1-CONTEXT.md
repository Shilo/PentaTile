# Phase 1.1 Context: PentaTile Rename + Penta Codename Establishment

**Status:** Inserted 2026-04-26 (between Phase 1 done, Phase 2 not started)

**Driver:** v0.2 adds a 5th archetype (OppositeCorners) and `tile_count: TileCountMode` mode FIVE — the project's identity is shifting from "4-tile autotiler" to "5-archetype autotiler." The name follows the identity. Coins "Penta" as the project's tileset codename, analogous to "Wang" and "Blob" — a short, descriptive, unowned label propagated through usage and a canonical definition page (not trademarked).

## Objective

Rename `Tetra` → `Penta` and `tetra` → `penta` project-wide BEFORE Phase 2 ships new files under the old name. Establish "Penta" as the official codename for the 5-archetype tileset format via a canonical README definition. Land it as one (or very few) atomic commits so Phase 2 starts in a fully Penta-named codebase.

## Risks / concerns

1. **Custom data layer keys silently orphan saved scenes** (CLAUDE.md Pitfall #3). `tetra_role` / `tetra_lock_rotation` in `tetra_tile_ground.tres` won't auto-migrate in Godot 4.6. Per the no-compat policy, one-time re-author of the demo TileSet is acceptable.
2. **Plugin folder rename via filesystem** — `.tscn` references `addons/tetra_tile/...` paths. Godot 4.6 UID system handles renames done THROUGH the editor; raw filesystem renames may break references. Use the editor for the move, OR run a UID refresh + manual fix-up, OR use the `mcp__godot__update_project_uids` tool.
3. **2,398 occurrences across 86 files — review-and-replace, not blind sed.** Edge case: `.planning/research/layouts/RPG_MAKER.md` may use "tetra" as generic gamedev folklore for 4-tile arrangements predating this project; those refs (if any) shouldn't rename. Check before each replace.
4. **GitHub rename requires user action.** Agent cannot rename the GitHub repo without credentials. The plan must call this out as a manual gate, with the local-side `git remote set-url` happening AFTER the GitHub rename completes.
5. **Claude memory migration must be paired with directory rename.** If one happens without the other, the next session starts with no memory of this project's accumulated context.
6. **Atomic commit discipline.** A 2,398-replace sweep that lands across multiple commits has windows where the codebase is half-renamed and won't compile/load. Aim for one commit, or very few commits each of which leaves the project working.

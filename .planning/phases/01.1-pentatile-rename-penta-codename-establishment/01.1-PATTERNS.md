### NO PRIOR ANALOG

This repo has no history of an addon-folder rename. Per CONTEXT.md Risk #2, three options are available — the planner must pick one:

1. **Editor-driven rename (RECOMMENDED):** open Godot 4.6, use the FileSystem dock's right-click → "Move/Rename" on each file. Godot 4.6 automatically:
   - Preserves the UID (file content stays the same)
   - Updates ext_resource paths in every `.tscn` / `.tres` / `.import` that referenced the old path
   - Updates `path="res://addons/tetra_tile/..."` strings throughout the project tree
   - This is the safest path; trust Godot's UID system.
2. **Raw `git mv` + manual fix-up:** rename via filesystem, then grep-and-replace every `addons/tetra_tile/` string in `.tscn` / `.tres` / `.import` files. Riskier — Godot 4.6 may regenerate UIDs on next project load if path/UID mismatch is detected.
3. **`mcp__godot__update_project_uids` tool:** if available in the agent environment, this is the surgical option.

**The `.import` files contain `source_file="res://addons/tetra_tile/..."`** — see `templates/tetra_horizontal.png.import` line 13:
